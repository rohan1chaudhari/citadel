# Support

## Where to ask for help
- Open a GitHub issue for bugs/features.
- Include logs, repro steps, and environment details.

## What to include in bug reports
1. Expected behavior
2. Actual behavior
3. Exact URL/route used
4. Logs (host + external app)
5. Steps to reproduce

## Quick diagnostics
- Host apps list: `GET /api/apps`
- External health:
  - French: `:4013/api/healthz`
  - Gym: `:4014/api/health`
  - Scrum: `:4015/api/healthz`
  - Smart Notes: `:4016/api/healthz`
